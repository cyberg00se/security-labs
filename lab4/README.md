# Lab4: Human-like password generator

## Part 1

1. Для каждого метода хеширования генерируется по 100К паролей. Пароли трёх видов: 
	- каждый двадцатый пароль (5К штук) делается рандомной длины (10-15) из рандомных символов и может содержать (не обязательно всё сразу) латиницу строчную/прописную, цифры и символы $%^&)><?_.,@
	- ещё 5К штук берутся из списка 100 самых популярных паролей
	- остальные 90К берутся из списка 100К самых популярных паролей

2. Первые 100К паролей хешируются c использованием MD5, размер хеша - 128 бит. Время не фиксировала, но относительно совсем мало.

3. Вторые 100К паролей хешируются c использованием bcrypt со значением cost=10. Готовый хеш содержит префикс с версией bcrypt (2b), значение cost (10), а также 22 символа соли (всего 128 бит) и 31 символ хеша пароля (192 бита), которые закодированы c помощью Radix-64.
