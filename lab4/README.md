# Lab4: Human-like password generator

## Part 1

1. Для каждого метода хеширования генерируется по 100К паролей. Пароли трёх видов: 
	- каждый двадцатый пароль (5К штук) делается рандомной длины (10-15) из рандомных символов и может содержать (не обязательно всё сразу) латиницу строчную/прописную, цифры и символы $%^&)><?_.,@
	- ещё 5К штук берутся из списка 100 самых популярных паролей
	- остальные 90К берутся из списка 100К самых популярных паролей

2. Первые 100К паролей хешируются c использованием MD5, размер хеша - 128 бит. Время не фиксировала, но относительно совсем мало.

3. Вторые 100К паролей хешируются c использованием bcrypt со значением cost=10. Готовый хеш содержит префикс с версией bcrypt (2b), значение cost (10), а также 22 символа соли (всего 128 бит) и 31 символ хеша пароля (192 бита), которые закодированы c помощью Radix-64.

Все сгенерированные штуки в папке results.

## Part 2

Всё в папке part2, использовался hashcat.

1. Первый файл weakHashes.csv выглядит как последовательность из 32 шестнадцатеричных цифр (16 байт), похоже на алгоритм MD5. Для восстановления паролей была проведена атака со словарём (в качестве словаря взят 1 миллион самых популярных паролей). 
	- Из 100К хешей в файле уникальными были 57027, из них удалось восстановить 42108 (73.84%). 
	- Ещё 14919 (26.16%) паролей таким способом восстановить не удалось.
	- Затраченное время - 1 секунда.

2. Второй взятый файл strongHashes.csv содержит хеши алгоритма bcrypt (колонка с отдельной солью была удалена). Для восстановления паролей была проведена атака с тем же словарём. 
	- Из 100К хешей в файле уникальными были все 100К (из-за использования соли), из них удалось восстановить 95827 (95.83%). 
	- Ещё 4173 (4.17%) паролей, возможно, можно было бы тоже восстановить, но hashcat наравлся на какой-то сложный пароль, которого, видимо, не было в словаре.
	- Поэтому атака была прервана спустя 3 часа и 17 минут после начала.

3. На других файлах с хешами MD5 пробовались атаки брутфорсом и с маской, но ои занимают намного больше времени. При этом чем длиннее пароль и чем больше возможный набор символов в нём, тем сложнее такое провернуть (или вообще невозможно).

Общие выводы: всё, что похоже на словарь, будет найдено легко (хороший алгоритм хеширования замедлит это дело, конечно, но всё равно недостаточно. только bcrypt с cost всего 5 как в файле из пункта 2 использовать не надо). При этом если пароль не совпадает со словарём, длинный и содержит, например 3 вида символов (строчные буквы, цифры и несколько других символов), то даже при использовании MD5 без соли докопаться до него будет сложнее.
